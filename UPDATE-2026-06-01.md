## 二、新增：macOS 本地代理自动切换机制

### 问题背景

之前 `~/.zshrc` 中硬编码了代理环境变量：

```bash
export ALL_PROXY="socks5://127.0.0.1:1080"
export HTTPS_PROXY="socks5://127.0.0.1:1080"
```

导致 VPN 断开后，所有网络请求仍往已关闭的 SOCKS5 端口发，造成：
- 国内网站访问极慢（等待超时才 fallback）
- 需要手动 `unset` 或删除 `.zshrc` 中的变量

### 解决方案

使用 zsh 的 `precmd` hook，在每次命令执行前自动检测 Shadowsocks 本地端口状态：

```zsh
# ~/.zshrc 中的实现

_auto_proxy() {
  if nc -z -w1 127.0.0.1 1080 2>/dev/null; then
    # 端口通 → VPN 连接中 → 设置代理
    if [[ -z "$ALL_PROXY" ]]; then
      export ALL_PROXY="socks5://127.0.0.1:1080"
      export HTTPS_PROXY="socks5://127.0.0.1:1080"
      export NO_PROXY="token-plan-cn.xiaomimimo.com,*.xiaomi.com,*.qq.com,*.wechat.com,localhost,127.0.0.1"
    fi
  else
    # 端口不通 → VPN 断开 → 清除代理
    if [[ -n "$ALL_PROXY" ]]; then
      unset ALL_PROXY HTTPS_PROXY HTTP_PROXY http_proxy https_proxy ALL_PROXY all_proxy NO_PROXY no_proxy
    fi
  fi
}
autoload -Uz add-zsh-hook
add-zsh-hook precmd _auto_proxy
```

### 工作流程

```
用户在终端执行任意命令
        ↓
precmd hook 触发 _auto_proxy()
        ↓
nc -z -w1 127.0.0.1 1080  (毫秒级端口检测)
        ↓
   ┌────┴────┐
   端口通      端口不通
   (VPN连着)   (VPN断开)
   ↓           ↓
检查 $ALL_PROXY  检查 $ALL_PROXY
是否已设置？      是否存在？
   ↓           ↓
  已设置→跳过    不存在→跳过
  未设置→设置    存在→清除
```

### 设计要点

1. **幂等检测**: `if [[ -z "$ALL_PROXY" ]]` / `if [[ -n "$ALL_PROXY" ]]` 避免重复操作
2. **性能**: `nc -z -w1` 超时 1 秒，实际本地检测通常毫秒级完成
3. **NO_PROXY 国内直连**: 包含 `*.xiaomi.com`, `*.qq.com`, `*.wechat.com` 等国内域名
4. **零手动操作**: 连 VPN 自动代理，断 VPN 自动直连，用户无感知

### 已知限制

- `source ~/.zshrc` 只注册 hook，不会立即执行一次检测，旧变量可能残留
- 解决：手动执行 `_auto_proxy` 或开新终端
- 每个终端窗口独立维护自己的环境变量状态

```
$ nc -z -w1 127.0.0.1 1080 && echo "端口通" || echo "端口不通"

ALL_PROXY=socks5://127.0.0.1:1080
HTTPS_PROXY=socks5://127.0.0.1:1080
NO_PROXY=token-plan-cn.xiaomimimo.com,*.xiaomi.com,*.qq.com,*.wechat.com,localhost,127.0.0.1
```
✅ 代理变量自动设置

```
$ nc -z -w1 127.0.0.1 1080 && echo "端口通" || echo "端口不通"
端口不通

$ unset ALL_PROXY HTTPS_PROXY ... && source ~/.zshrc && env | grep -i proxy || echo "无代理变量"
无代理变量
```
✅ 代理变量自动清除
